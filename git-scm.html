Why git?

  • The name is intended
  • Distributed
    • no need for a connection to work
    • integrated backup
    • own playground
  • Free

Learn to ... fall

  • Don't panic, your data is probably there
  • Don't push
  • Don't force
  • Don't gc
  • Don't delete
  • When in doubt, create a copy of the folder.

Keep in mind...

  • It's not magic
  • git is composed of simple building blocks
  • git is not smart

Simple workflow

  1. Create/modify files
  2. Add them to staging (index)
  3. Commit it with a message
  4. GOTO 1

Working with branches

  • Branching is copying, but smarter.
  • A branch is a copy of entire project.
  • To save space only the differences are stored.

How to branch?

  1. Create a branch
  2. Switch to branch
  3. Create/modify files
  4. Add them to staging (index)
  5. Commit it with a message
  6. GOTO 3

Incorporating changes

Use git merge

Working with a remote

  • When you clone, there are two repositories
  • Branches between repositories can be connected (tracking)
  • Your local repository keeps references for remote branches

Remote commands

  • git clone
  • git fetch
  • git pull
  • git push
